## Домашняя работа №1

Домашняя работа №1 состоит из двух мини-задач.

### Мини-задача 1: Zipf, предобработка и интенты

Файл: `первая домашняя работа/первая мини-задача/homework_task1.py`

Что делает скрипт:

1. Загружает фрагмент «Белых ночей» (5 страниц) и строит анализ по закону Ципфа.
2. Считает длинный хвост словаря (hapax и слова с частотой `<= 3`).
3. Сравнивает варианты предобработки:
   - raw,
   - lowercase,
   - замена `ё -> е`,
   - удаление коротких слов,
   - лемматизация (`pymorphy3`, если установлен).
4. Определяет интенты через `spaCy Matcher`:
   - базовые (`greeting`, `goodbye`, `ask_weather`, `ask_time`),
   - добавленные (`ask_recommendation`, `express_gratitude`).

Результат выполнения:

- таблицы частот и сравнения словарей,
- графики (Zipf + размер словаря по шагам предобработки),
- распознанные интенты для тестовых фраз.

### Мини-задача 2: токены/стеммы/леммы + NER/POS/DEP

Файл: `первая домашняя работа/вторая мини-задача/homework_task2.py`

Что делает скрипт:

1. Загружает русский художественный текст (фрагмент «Белых ночей»).
2. Сравнивает частоты и словари для:
   - токенов,
   - стемм (`nltk` SnowballStemmer),
   - лемм (`pymorphy3`).
3. Считает долю длинного хвоста (`f <= 3`) для каждого представления.
4. Для 5 предложений показывает:
   - POS/DEP/NER из `spaCy`,
   - сравнение лемм `spaCy` и `pymorphy3`.
5. Извлекает персонажей (NER `PER`) и считает частоты упоминаний.
6. Ищет предложения с приветствиями двумя способами:
   - через `regex`,
   - через `spaCy Matcher`.

Результат выполнения:

- сравнительные таблицы,
- графики по словарям и хвосту,
- список персонажей и найденные приветствия.

## Домашняя работа №2

Основной файл: `вторая домашняя работа/МПИ_25_1_2_Исмаилова_Наргиз_Domain_Shift.ipynb`

Подробное текстовое описание: `вторая домашняя работа/README`

Работа состоит из двух частей.

### Часть A: Domain Shift и распределения

1. Используется датасет `data-silence/rus_news_classifier`.
2. Сравниваются два домена: `politics` и `sports`.
3. Считаются log-odds токены (в обе стороны).
4. Анализируются распределения:
   - длины документов,
   - Zipf-кривые.
5. Проверяется влияние предобработки (стоп-слова, лемматизация).

### Часть B: Authorship Attribution (Naive Bayes)

1. Поэтический корпус делится на чанки.
2. Обучается `MultinomialNB` для определения автора.
3. Оцениваются `accuracy`, `macro F1`, `confusion matrix`.
4. Анализируются важные токены по разнице лог-вероятностей.
5. Сравниваются варианты с лемматизацией и n-граммами.
6. Отдельно проверяется domain shift (длинные тексты -> короткие).
